# Example: Teaser Post

> This teaser was produced for the "Fat Marker / Agentic Development" article.
> It demonstrates the teaser formula: surprising stat → brief expansion → what the article covers → link CTA.

---

We were already building the wrong things 80% of the time. Agentic development makes that worse, not better.

Pendo's data is clear: 80% of features are rarely or never used. Only 12% generate 80% of daily usage. The bottleneck was never "how fast can we code it?" — it was "do we understand the problem well enough to build the right thing?"

AI agents removed the cost of building. They didn't remove the cost of building the *wrong thing*. That cost is the same as it ever was — confused users, wasted cycles, features nobody asked for.

The fix isn't new. It's old. Fat Marker sketches, design thinking, first principles — the fundamentals of understanding the problem before solving it.

I wrote about what this looks like in practice and why the teams that win with AI won't be the fastest shippers. Link in comments.
